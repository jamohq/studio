## Show HN (Hacker News)

**Title:** Show HN: Jamo Studio -- open-source desktop IDE with visual canvas, AI chat, and git built in

**Text:**

Hi HN, I built Jamo Studio -- a local-first desktop app that combines a visual canvas (Excalidraw), rich text editor, code editor (CodeMirror), AI chat (Claude), terminal, and git version control into one workspace.

The idea: instead of jumping between Figma, VS Code, a terminal, and ChatGPT, you work in one app where everything is connected. You describe your app with drawings and structured text (purpose, tech stack, brand guidelines, user stories), and the AI can use all of that context when generating or editing code. Every AI action is auto-committed with tags so you can always see what changed and revert with one click.

Architecture: Electron + React frontend, Go gRPC backend for workspace ops and terminal. Everything runs locally -- no cloud, no accounts, no telemetry.

Tech: TypeScript, React, Electron, Go, gRPC, Excalidraw, CodeMirror 6, TipTap, xterm.js

Open source (AGPL-3.0): https://github.com/jamohq/studio

I'd love feedback on the concept and UX. What would make this useful for your workflow?

## Reddit (r/webdev, r/sideproject)

**Title:** I built an open-source desktop IDE that combines visual design, AI chat, code editor, terminal, and git -- all in one app

**Body:**

I've been working on Jamo Studio, an open-source desktop app for building software.

The problem I kept hitting: my workflow was scattered across too many tools. Figma for mockups, VS Code for code, terminal for running things, ChatGPT in a browser tab, git in yet another terminal. None of them know about each other.

Jamo Studio puts it all in one place:

- **Visual canvas** (Excalidraw) for wireframes and flows
- **Rich text sections** for project context (purpose, tech stack, brand, user stories)
- **AI chat** (Claude) that can see your files AND your designs
- **Code editor** (CodeMirror 6, vim mode)
- **Built-in terminal** (Go PTY backend)
- **Git source control** with auto-commits for AI actions

The AI isn't a black box -- every generation is tracked as a "run" with inputs, outputs, and history. You can re-run with notes ("use CSS Grid instead") or one-click revert.

Everything runs locally on your machine. No cloud. No accounts. AGPL-3.0.

GitHub: https://github.com/jamohq/studio

Would love to hear what you think. What features would make this useful for you?

## Product Hunt

**Tagline:** Design, code, and ship -- all in one local-first desktop app.

**Description:**

Jamo Studio is an open-source desktop IDE that combines a visual canvas, AI chat, code editor, terminal, and version control into one workspace.

**The problem:** Building software means juggling Figma, VS Code, a terminal, ChatGPT, and git. None of these tools share context with each other.

**The solution:** One app where everything is connected. Draw wireframes on a canvas. Describe your app in structured sections. Chat with AI that can see your designs, your code, and your terminal. Every AI-generated change is auto-committed and reversible.

**Key features:**
- Visual canvas (Excalidraw) for wireframes and flows
- AI chat (Claude) with full project context
- Code editor (CodeMirror 6) with vim mode
- Built-in terminal with Go PTY backend
- Git version control with auto-commits and tagged history
- Rich text sections for project documentation
- 100% local -- no cloud, no accounts, no telemetry

**Built with:** Electron, React, TypeScript, Go, gRPC

**Open source:** AGPL-3.0
